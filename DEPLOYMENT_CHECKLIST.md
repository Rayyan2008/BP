# Being Personalised - Deployment Checklist

## Pre-Deployment Phase

### 1. Build Verification
- [x] Project builds successfully (`pnpm build`)
- [x] No TypeScript errors
- [x] No ESLint errors
- [x] All imports are valid
- [x] No hydration issues detected
- [x] App Router conventions implemented correctly

### 2. Environment Setup
- [ ] Copy `.env.example` to `.env.local`
- [ ] Fill in all required environment variables:
  - [ ] `NEXT_PUBLIC_SUPABASE_URL`
  - [ ] `NEXT_PUBLIC_SUPABASE_ANON_KEY`
  - [ ] `SUPABASE_SERVICE_ROLE_KEY`
  - [ ] `NEXT_PUBLIC_WHATSAPP_NUMBER`
  - [ ] `NEXT_PUBLIC_PHONE_NUMBER`
  - [ ] `NEXT_PUBLIC_EMAIL_ADDRESS`
- [ ] Verify no sensitive data in `.env.local` is committed

### 3. Admin System Setup
- [ ] Admin login page accessible at `/admin/login`
- [ ] Admin password set to: `SapotoInfosys`
- [ ] Admin dashboard accessible at `/admin/dashboard`
- [ ] Test admin login/logout functionality
- [ ] Verify enquiry form data appears in admin panel
- [ ] Test status update functionality
- [ ] Test internal notes functionality

### 4. Contact Features Verification
- [ ] Floating WhatsApp button appears on all pages
- [ ] WhatsApp button opens correct chat with pre-filled message
- [ ] Mobile quick contact bar appears on mobile devices
- [ ] All contact buttons (WhatsApp, Phone, Email, Instagram) are functional
- [ ] Contact form fields all present and working
- [ ] Form submission saves to admin panel
- [ ] Success message displays after form submission

### 5. Database/Backend Verification
- [ ] Supabase project configured
- [ ] Environment variables point to correct Supabase instance
- [ ] Enquiries are persisted correctly
- [ ] Admin data updates persist across sessions

## Vercel Deployment Phase

### 1. Vercel Configuration
- [ ] Project connected to Vercel
- [ ] Git repository connected
- [ ] Build command: `pnpm build`
- [ ] Start command: `pnpm start`
- [ ] Install command: `pnpm install`

### 2. Environment Variables on Vercel
- [ ] All `.env.example` variables added to Vercel project settings
- [ ] `NEXT_PUBLIC_` prefixed variables set for client-side use
- [ ] Verify no TypeScript errors in Vercel build logs
- [ ] Check production build output in Vercel dashboard

### 3. Domain Configuration
- [ ] Domain DNS records updated to point to Vercel
- [ ] SSL certificate auto-generated by Vercel
- [ ] HTTPS enforced (automatic with Vercel)
- [ ] Redirect from www to non-www (or vice versa) configured
